--- layout: default title: How to Report a Bug ---

So you found a bug and want to report it? Here's what you need to know.

It should be no secret that I rely on you to report bugs, so I can improve your experience.
There are two ways to report a bug:
  1. Via GitHub
  2. Via NexusMods

Do not

  • Ask for features in a bug report.
    Please only use special GitHub Issues for that.
  • Ask questions in a bug report.
    Please use the dedicated channels (Discord, NexusMods Posts) or have a look at the FAQ.

The title

The title is the very first thing I see and will reference multiple times. Thus, it needs to describe the core of your issue in a short and sensible fashion.
Pleas do not put the title as "Manager not work", "Program bad", ect. This is not helpful at all and I can't improve the tool with that.

What to include

This should go without saying but please don't make the description just your title again.

If you are reporting a crash or an error please include the log file in question. You will find your logs in the same place where you put your manager executable.
In any case, describe what you were doing up until your issue occurred.
It is very important that you give me every bit of context you can, even if it's mod specific. (I won't judge you)

You don't need to include any contact information. GitHub and NexusMods will take care of that. I will never ask you for personal information.
Also, you should know better than to post personal information online on demand.
